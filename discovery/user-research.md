# User Research

## Target Audience

**Primary User:** Elderly Indian parents (age 60+)
- Not comfortable typing on mobile keyboards
- Hindi is their primary language
- Familiar with voice/speech but not app navigation
- May have limited eyesight or dexterity

## User Needs

| Need | Description |
|------|-------------|
| Voice-first | All interactions should happen through voice — no typing required |
| Hindi language | UI text, voice prompts, and responses must be in Hindi |
| Large touch targets | Buttons and text must be large and easy to tap |
| Minimal navigation | No complex menus, tabs, or settings |
| Forgiving UX | App should handle unclear speech gracefully and ask again |

## User Context

- Used primarily on Android mobile browsers
- Network may be slow or intermittent
- Users may speak natural Hindi with regional accents
- Users often use traditional measurement terms (पाव, डेढ़ पाव, etc.)
- Users may say grocery items in mixed Hindi/Hinglish

## Key User Flows

1. Open app → Sign in or skip → Create list by voice → Add items by voice → Done
2. Return to app → Open existing list → Mark items while shopping
3. Review all lists

## Pain Points Avoided

- No keyboard input required
- No complex registration flow (skip login available)
- No pagination or filtering
- No settings to configure
